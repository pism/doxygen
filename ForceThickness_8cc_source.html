120 /*!
121 If `-force_to_thickness_file` `foo.nc` is in use then vthktarget will have a target ice thickness
122 map. Let \f$H_{\text{tar}}\f$ be this target thickness,
123 and let \f$H\f$ be the current model thickness. Recall that the mass continuity
124 equation solved by GeometryEvolution is
125  \f[ \frac{\partial H}{\partial t} = M - S - \nabla\cdot \mathbf{q} \f]
126 and that this procedure is supposed to produce \f$M\f$.
127 In this context, the semantics of `-force_to_thickness` are that \f$M\f$ is modified
128 by a multiple of the difference between the target thickness and the current thickness.
129 In particular, the \f$\Delta M\f$ that is produced here is
130  \f[\Delta M = \alpha (H_{\text{tar}} - H)\f]
131 where \f$\alpha>0\f$ is determined below. Note \f$\Delta M\f$ is positive in
132 areas where \f$H_{\text{tar}} > H\f$, so we are adding mass there, and we are ablating
133 in the other case.
134 
135 Let \f$t_s\f$ be the start time and \f$t_e\f$ the end time for the run.
136 Without flow or basal mass balance, or any surface mass balance other than the
137 \f$\Delta M\f$ computed here, we are solving
138  \f[ \frac{\partial H}{\partial t} = \alpha (H_{\text{tar}} - H) \f]
139 Let's assume \f$H(t_s)=H_0\f$. This initial value problem has solution
140 \f$H(t) = H_{\text{tar}} + (H_0 - H_{\text{tar}}) e^{-\alpha (t-t_s)}\f$
141 and so
142  \f[ H(t_e) = H_{\text{tar}} + (H_0 - H_{\text{tar}}) e^{-\alpha (t_e-t_s)} \f]
143 
144 The constant \f$\alpha\f$ has a default value `pism_config:surface.force_to_thickness.alpha`.
145 
146 The next example uses files generated from the EISMINT-Greenland experiment;
147 see the corresponding chapter of the User's Manual.
148 
149 Suppose we regard the SSL2 run as a spin-up to reach a better temperature field.
150 It is a spinup in which the surface was allowed to evolve. Assume the
151 early file `green20km_y1.nc` has the target thickness, because it essentially
152 has the input thickness. This script adds a 500 a run, to finalize the spinup,
153 in which the ice sheet geometry goes from the the thickness values in
154 `green_ssl2_110ka.nc` to values very close to those in `green20km_y1.nc`:
155 \code

228 void ForceThickness::adjust_mass_flux(double time,
229  const IceModelVec2S &ice_thickness,
230  const IceModelVec2CellType &cell_type,
231  IceModelVec2S &result) const {
232 
233  if (time < m_start_time) {
234  return;
235  }
236 
237  m_log->message(5,
238  " updating surface mass balance using -force_to_thickness mechanism ...");
239 
240  double ice_density = m_config->get_number("constants.ice.density");
241 
242  IceModelVec::AccessList list{&cell_type, &ice_thickness,
243  &m_target_thickness, &m_ftt_mask, &result};
244 
245  for (Points p(*m_grid); p; p.next()) {
246  const int i = p.i(), j = p.j();
247 
248  if (m_ftt_mask(i,j) > 0.5 and cell_type.grounded(i, j)) {
249  if (m_target_thickness(i,j) >= m_ice_free_thickness_threshold) {
250  result(i,j) += ice_density * m_alpha * (m_target_thickness(i,j) - ice_thickness(i,j));
251  } else {
252  result(i,j) += ice_density * m_alpha * m_alpha_ice_free_factor * (m_target_thickness(i,j) - ice_thickness(i,j));
253  }
254  }
255  }
256  // no communication needed
257 }

290 /*!
291 The timestep restriction is, by direct analogy, the same as for
292  \f[\frac{dy}{dt} = - \alpha y\f]
293 with explicit (forward) Euler. If \f$\Delta t\f$ is the time step then Euler is
294 \f$y_{n+1} = (1-\alpha \Delta t) y_n\f$. We require for stability that
295 \f$|y_{n+1}|\le |y_n|\f$, which is to say \f$|1-\alpha \Delta t|\le 1\f$.
296 Equivalently (since \f$\alpha \Delta t>0\f$),
297  \f[\alpha \Delta t\le 2\f]
298 Therefore we set here
299  \f[\Delta t = \frac{2}{\alpha}.\f]
300  */
301 MaxTimestep ForceThickness::max_timestep_impl(double my_t) const {
302  double max_dt = 2.0 / m_alpha;
303  MaxTimestep input_max_dt = m_input_model->max_timestep(my_t);
304 
305  return std::min(input_max_dt, MaxTimestep(max_dt, "surface forcing"));
306 }
